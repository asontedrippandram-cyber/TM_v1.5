## 3. 五维度进度模型

### 3.1 维度定义
每个子特性都有5个测试维度：
1. **内部台架测试** (benchTest)
2. **内部实车测试** (vehicleTest)
3. **集成冒烟测试** (smokeTest)
4. **实车点检** (pointCheck)
5. **整车E2E验证** (e2eTest)

### 3.2 进度构成
每个维度的进度 = **用例执行进度** × 50% + **手工任务进度** × 50%

**用例执行进度**：
- 通过 CSV 文件导入
- 格式：`subVehicleId, subFeatureId, dimension, passedCases, totalCases`
- 可对接外部 API 获取

**手工任务进度**：
- 进度滑块：0% - 100%
- 富文本说明：用户填写任务进展备注

### 3.3 并行处理
- 5个维度可以同时展开和编辑
- **无顺序锁定** - 所有维度始终可编辑
- 维度之间相互独立

### 3.4 CSV 导入
点击"导入用例"按钮，选择 CSV 文件即可导入用例进度数据。

**demo.csv 示例格式：**
```csv
subVehicleId,subFeatureId,dimension,passedCases,totalCases
sv1,sf1,benchTest,8,10
sv1,sf1,vehicleTest,6,10
sv1,sf1,smokeTest,0,5
...
```

## 4. 每车型独立特性

### 4.1 数据结构
车型可以独立定义自己的特性和子特性，未定义的车型使用公共特性。

```javascript
subVehicles: [
  {
    id: 'sv1',
    name: 'F2D',
    milestones: {
      commercial: '20260630',
      epRelease: '20260430',
      ppRelease: '20260531'
    },
    // 可选：独立定义特性，不定义则使用公共特性
    features: [
      { id: 'f1', name: '互动游戏', subFeatures: [...] }
    ]
  }
]
```

### 4.2 回退机制
- 如果子车型没有定义 `features`，使用 `CONFIG.features`（公共特性）
- 确保所有车型都有特性可用

## 5. 多里程碑时间轴

### 5.1 里程碑定义
每个子车型支持多个关键时间节点：
- **EP 发布** (epRelease)：工程样机发布
- **PP 发布** (ppRelease)：预生产发布
- **商用** (commercial)：正式商用

### 5.2 时间轴显示
- 时间线显示从项目启动到商用日期的进度
- 里程碑标记显示在时间线上
- 已过的里程碑显示为灰色

## 7. 交互流程

1. 打开页面 → 顶部显示所有车型风险卡片
2. 点击"导入用例" → 选择 CSV 文件导入用例进度
3. 点击任意卡片 → 该车型高亮，下方展开子特性列表
4. 点击子特性 → 展开五维度详情，**所有维度可并行编辑**
5. 编辑维度进度 → 调整手工滑块，填写说明，保存

## 8. 数据结构

### featureDimensions (localStorage: carTestProjects)
```javascript
"subVehicleId-subFeatureId": {
  "dimensionKey": {
    manualProgress: number,  // 手工进度 0-100
    manualText: string       // 手工任务说明
  }
}
```

### caseProgressData (localStorage: carTestCaseProgress)
```javascript
"subVehicleId-subFeatureId-dimension": {
  passed: number,  // 通过用例数
  total: number,   // 总用例数
  progress: number // 进度百分比
}
```
